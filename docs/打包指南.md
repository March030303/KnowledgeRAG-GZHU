## 二、本次新增：桌面端打包

### 方案 A：Tauri 桌面端（轻量，推荐）

**优势：** 体积小（3-5MB 安装包）、性能高、安全沙箱

#### 快速开始

```bash
# 1. 开发预览（Tauri 启动本地 Vite 服务）
cd RagFrontend
npm run tauri:dev

# 2. 构建 Windows 安装包
npm run tauri:build
# 输出：src-tauri/target/release/bundle/nsis/*.exe
```

#### 配置文件说明

| 文件                                  | 说明                                       |
| ------------------------------------- | ------------------------------------------ |
| `src-tauri/tauri.conf.json`           | Tauri 应用配置（窗口大小、图标、打包目标） |
| `src-tauri/Cargo.toml`                | Rust 依赖和项目元数据                      |
| `src-tauri/src/lib.rs`                | Rust 主进程入口                            |
| `src-tauri/capabilities/default.json` | 前端可调用的系统权限                       |

#### 注意事项

- Tauri 打包仅封装**前端页面**，后端需独立运行（Docker 或 PyInstaller）
- 首次构建需下载 Rust 工具链（约 1-2GB），后续构建会缓存
- Windows 打包需安装 [Rust](https://rustup.rs/) 和 [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022)

### 方案 B：PyInstaller 后端独立打包

**用途：** 将 FastAPI 后端打包为可执行文件，无需 Python 环境即可运行

#### 快速开始

```bash
# 1. 安装 PyInstaller
pip install pyinstaller

# 2. 打包后端（项目根目录下执行）
pyinstaller rag_backend.spec

# 3. 输出目录
#    Windows: dist/rag_backend.exe
#    Linux/macOS: dist/rag_backend
```

#### 环境要求

| 依赖           | 说明                          |
| -------------- | ----------------------------- |
| Python 3.11+   | 建议使用虚拟环境              |
| Windows 64-bit | 当前 spec 配置为 Windows 平台 |

#### 打包后使用

```bash
# 1. 复制到部署目录
cp dist/rag_backend.exe ./deploy/

# 2. 准备依赖目录
mkdir -p deploy/data
mkdir -p deploy/metadata
mkdir -p deploy/user_avatars
mkdir -p deploy/knowledge_base
mkdir -p deploy/local-KLB-files

# 3. 配置环境变量（创建 .env 或 set 环境变量）
set DB_HOST=localhost
set DB_PORT=3306
set DB_USER=root
set DB_PASSWORD=xxx
set DB_NAME=rag_user_db
set REDIS_URL=redis://localhost:6379/0
set OLLAMA_BASE_URL=http://localhost:11434
set JWT_SECRET=your_jwt_secret

# 4. 启动（需要 MySQL 和 Redis 已运行）
rag_backend.exe
# 默认监听 http://127.0.0.1:8000
```

### 方案 C：前后端一键启动包（推荐组合）

结合 Tauri + PyInstaller，用户双击即可启动完整系统：

```
KnowledgeRAG-Setup/
├── rag_backend.exe          # PyInstaller 打包的后端
├── run.bat                  # 一键启动脚本
├── .env.example             # 环境变量模板
├── mysql/                   # MySQL 便携版（可选）
├── redis/                   # Redis 便携版（可选）
└── models/                  # Ollama 模型目录（可选）
```

#### `run.bat` 脚本

```bat
@echo off
chcp 65001 >nul
title KnowledgeRAG 后端服务

echo ============================================
echo   KnowledgeRAG 后端服务启动中...
echo ============================================

:: 检查 Python 环境
python --version >nul 2>&1
if errorlevel 1 (
    echo [错误] 未检测到 Python 环境，请安装 Python 3.11+
    pause
    exit /b 1
)

:: 启动后端
cd /d "%~dp0"
start "KnowledgeRAG" rag_backend.exe

:: 等待后端启动
timeout /t 3 /nobreak >nul

:: 检查后端是否运行
curl -s http://127.0.0.1:8000/docs >nul 2>&1
if errorlevel 1 (
    echo [警告] 后端可能未正常启动，请检查端口 8000 是否被占用
) else (
    echo [成功] 后端已启动，访问 http://127.0.0.1:8089 使用前端
    echo [成功] API 文档 http://127.0.0.1:8000/docs
)

pause
```

## 三、Docker 部署（生产环境，推荐）

### GPU 加速（可选）

取消 `docker-compose.yml` 中 Ollama 的 GPU 配置注释：

```yaml

    resources:
      reservations:
        devices:
          - driver: nvidia
            count: all
            capabilities: [gpu]
```

## 七、常见问题

### Q: Tauri 打包体积多大？

A: Tauri 本身约 3-5MB，比 Electron（~80MB）小很多。前端资源会被压缩，整体安装包约 20-50MB。

### Q: PyInstaller 打包后端体积多大？

A: 包含所有 Python 依赖约 500MB-1GB（取决于依赖数量）。比 Docker 镜像大，但无需安装 Docker。

### Q: 后端需要哪些外部服务？

A: MySQL + Redis（必须）+ Ollama（可选，用于本地模型）

### Q: 如何自定义图标？

A: 将图标文件放入 `src-tauri/icons/` 目录，Tauri 会自动使用。支持 .ico（Windows）、.icns（macOS）、.png 格式。
